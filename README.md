# Frontend Learning

## Overview

This repository is a **learning-first workspace** for building strong foundations in **frontend development** — starting from core web technologies (**HTML, CSS, JavaScript**) and gradually expanding to modern frontend concepts and frameworks.

It is intentionally structured for experimentation, iteration, and concept-level understanding rather than polished, portfolio-ready projects.

The goals of this repository are to:

* Build solid fundamentals in core web technologies
* Understand how browsers, the DOM, and UI rendering work
* Practice frontend concepts through small, focused demos
* Create a long-term reference for frontend fundamentals


## Repository Philosophy

* This repo prioritizes **learning and clarity over completeness**
* Code here may be rough, repetitive, or exploratory
* Each folder represents a **conceptual milestone** in frontend development
* Larger, refined, or showcase-worthy projects should live in **separate repositories**

This repository complements, but does not replace:

* Backend or scripting language repositories
* DSA and competitive programming repositories
* Portfolio or production-ready frontend projects

## How to Use This Repository

1. Pick a topic folder (e.g., `01-core-web/css` or `02-browser/dom`)
2. Build a minimal HTML/CSS/JS demo
3. Document:

   * What the concept is
   * Why it exists
   * Common pitfalls and observations
4. Commit frequently

Each subfolder should ideally contain:

* One or more `.html` files
* Associated `.css` / `.js` files
* A short `README.md` explaining the concept


## Frameworks and Tooling

Frameworks (such as React) and tooling are **intentionally deferred** until core concepts are well understood.

When introduced, frameworks will:

* Live under `05-frameworks/`
* Be treated as an abstraction layer on top of core web concepts
* Focus on understanding *why* they exist, not just *how* to use them


## What This Repository Is NOT

* A portfolio
* A framework showcase
* A production-ready codebase

When a demo grows beyond learning scope, it should be:

* Refactored
* Extracted
* Promoted to its own repository


## Progress Tracking

Progress is tracked through:

* Folder expansion
* README notes
* Commit history

This approach favors **visible learning over artificial completeness**.
